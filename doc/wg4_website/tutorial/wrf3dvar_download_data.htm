a. Downloading Test Data

This page describes how to access test datasets required to run WRF 3D-Var. If you don't know anything about 3D-Var, you should first read the Introduction section.

i. Required Data

The 3D-Var system requires three input files to run: a) The first guess – a) WRF input format file output from either the SI (cold-start) or WRF itself (warm-start), b) Observations (in BUFR or ASCII little_r format), and c) A background error statistics file (containing background error covariances currently calculated via the “NMC-method"). The use of these three data files in 3D-Var is described later in the tutorial.

The following table summarizes the above info:

Filename

Created By

Used In

fg

SI / WRF

3D-Var

ob

Observation Preprocessor

3D-Var

be

NMC_STATS code

3D-Var

In the online tutorial, example ob, fg, and be files are given. In your own work after the tutorial, you will need to create the input data sets yourselves.

ii. Downloading Test Data

In the oniline tutorial, we will store data in a directory defined by the environment variable $DAT_DIR and run 3D-Var in a directory defined by the environment variable $RUN_DIR. It is OK to set them equal if you wish. These directories can be in any location that can be read and written to from your current machine and which have at least 200MB of memory available (for this test case, may be more for other cases).  To set these variables, type

setenv DAT_DIR dat_dir

setenv RUN_DIR run_dir

where dat_dir and run_dir are your chosen directories. Create these directories (if they don’t already exist) and type

cd $DAT_DIR

Download the tutorial test data from the web-page for our standard case (3rd August 2003). Once you have downloaded the wrf3dvar_data.tar.gz file to $DAT_DIR, then extract it by typing

gunzip wrf3dvar_data.tar.gz

tar -xvf wrf3dvar_data.tar

You should then see the be, fg, ob and ob.little_endian data files in directory $DAT_DIR. Two observation files are supplied: ob for big_endian machines (e.g. IBM, SGI) and ob.little_endian for little endian machines (e.g. DEC, Linux). To save space type

rm wrf3dvar_data.tar
