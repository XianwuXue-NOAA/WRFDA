d. 3D-Var – Run Case Study

In this section, you will learn how to run 3D-Var using observations and a first guess from an actual operational system.

By this stage you have successfully created the three input files (first guess fg, observation ob and background error statistics be in directory $DAT_DIR) required to run 3D-Var. Also, you have successfully downloaded and compiled the 3D-Var code. If this is correct, we are ready to learn how to run 3D-Var. If not, then you’ll need to return to the 3D-Var Tutorial Page to complete stages a. to b.

Graphics: Not currently available – will be added later.

i. The Case

The data for this case is valid at 00Z 3rd August 2002. The first guess comes from NCEP’s global analysis, passed through the WRF Standard Initialization and “real” packages. All available observations valid within 90 minutes of analysis time are available.

ii. Required 3D-Var namelist changes

You will run 3D-Var using the script DA_Run_3DVAR.csh found in subdirectory wrf3dvar/run. Open this script with your favorite editor, and spend some time browsing it to accustom yourself with the layout. Don’t despair, in this tutorial we will not be changing too much in here. See b. Setting Up 3D-Var for more information on the various namelist options.

In your first experiment, the only changes you need to make to the DA_Run_3DVAR.csh script are to specify the DAT_DIR and RUN_DIR environment variables. Optionally, you can also change the RUN_ID variable. The script will then create a new subdirectory ${RUN_DIR}/${RUN_ID} in which 3D-Var will run - all output is directed there.

iii. Run 3D-Var

Once you have set the necessary environment variables, run 3D-Var for this case study by typing

DA_Run_3DVAR.csh

in the wrf3dvar/run subdirectory.

Successful completion of the job results in a number of output diagnostics files in the ${RUN_DIR}/${RUN_ID} subdirectory. The various textual diagnostics output files will be explained in the next section (3D-Var – Diagnostics). Here, we merely wish to run 3D-Var for this case.

iv. View 3D-Var Analysis Increments

To view the 3DVAR analysis results: wrf_3dvar_output, please download a simple utility View_analysis

In order to give yourself more experience with some of the 3D-Var options, try rerunning 3D-Var with different namelist.3dvar options. For example,

·          Withholding particular observation types by setting e.g. DA_USE_SYNOPOBS = .FALSE. (ignores surface SYNOP observations).

·          Making 3D-Var’s convergence more stringent by reducing the value of DA_EPS0 to e.g. 0.001. If 3D-Var hasn’t converged by the maximum number of iterations (DA_NTMAX) then you may need to increase DA_NTMAX.

As before, you may wish to change RUN_ID if you want to save all the outputs.
