1. Review of current developments (from WRF-Var Development table):

Update on KMA project:

 

Wei informed that global 3dvar has been updated for MPP. Rizvi tested the updated code on “cape” and IBM (bluesky). Multi processor run produced similar results. Since on IBM, T213 code cannot run using single processor so the muti-processor run results could not be compared with single processor run. However, similar exercise was performed for T63 and it produced same results.

 

Mr. Shin’s changes for “global converter” part have been integrated with global 3dvar code. The code compiles on CRAY and both Wei and Shin are testing it on CRAY.

 

Rizvi will test end-to-end global 3dvar run at T213 resolution on “Dec” and IBM machines. Wei will run on CARY-X1 and the results will be compared.

The convergence test with full global dataset for a particular case (12 UTC, 27 October, 2004) will be performed by increasing inner loop limit to 1000 and various diagnostics will be studied.

 

Wei informed that the global 3dvar code sent to Mi-Seon includes TC_Bogus & global changes. It does not include the “gen_be” part. Mi-Seon has already compiled this code successfully on CRAY machine with KMA. For MPP version, she has been advised to update namelist with nproc_y = 1.

 

Before 15th April, 2005 Wei has to merge Mi-Seon changes in global 3dvar for final version to be delivered to KMA. Obs preporcessor also needs to be tested along with global 3dvar.

 

Update on Background error code:

 

Dale informed that new BE code (gen_be) has been tested for NMC method in AMPS region. However, few changes needs to be made before its final delivery to for KMA. Currently there are six bins-type. In case of bins made with height, in certain cases, there may not be any point in some bins. Thus lat/model levels approach needs to be considered.

For KMA global, using NMC method BE-stats looks ok and the code runs very fast. For regional NMC method, scale length part is very slow. There are some problems for running gen_be code with ensembale method.

 

It was noticed that the rejection in wrf-3dvar for surface data is almost 90% and the biases are also very large specially, for the temperature. For MODIS rejection is about 10% but RMSE is very large (10 mps). Since the V-error is large than U-error indicating there is some problem with map projection and thus needs investigation). Dale proposed to add various diagnostics in “da_3dvar/utl” directory for tuning background errors and scale length parameters using Hollingworth approach.

 

Yong-Run tried three different formulations for computing innovation vector of Total Zenith delay, using vertical integration and surface pressure. All these methods are giving different results, which needs investigation. However, all the methods passed adjoint test.

 

Wrf-3dvar for giving separate treatment to Polar and Geostationary atmospheric motion vectors (AMV), developed by Rizvi needs to be reviewed.

 

4DVAR Development:

2. Project Spotlight: ?
